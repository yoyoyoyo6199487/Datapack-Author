ここでは簡単なコマンドや応用などを書いていきます

第一章 : execute
  これに関してはやれることが多すぎます
  僕だってこれを全部理解できてるか...w
  
  やっぱりexecuteと言えば as や　at が使われやすいような気がします
  1 : execute as とは
    書き方
      execute as @aまたは@eまたは@pまたは@r run
    どういう意味?
      指定したセレクターに強制的にコマンドを実行させるいうイメージでも大丈夫です[
      たとえば
      execute as @e[type=armor_stand] run kill なら
      "すべての防具立て" に KILLコマンドを実行させます
      killコマンドでセレクターが描かれてないということは
      自分をkillするというコマンドになります
      この "kill" の部分を give @s diamond に変更した場合
      自分にダイヤモンドを与えるコマンドになります
  2 : execute at とは
      これに関しては使わなくても大丈夫な場合があります
      そして僕は説明が下手なのでこういうややこしいやつはうまく説明できない可能性が高いです
    書き方
      execute at @aまたは@eまたは@pまたは@r run
    どういう意味?
      まず execute as は 自分を中心ですが 位置を中心にしているわけではありません
      もちろん自分が実行してるので座標は自分の位置に設定することは可能ですが
      コマンドの位置を自分以外の座標に設定したい場合
      例えば 自分を中心にしたい　でも　位置は違うプレイヤーの位置で実行したい!
      ってなった時
      そういう場合に使うのが execute at です
      ややこしいですね
      とにかく
      execute at は実行する位置をエンティティの位置で設定するものなので
      数字を入れてもエラーが発生します
      でも座標で設定したい!
      ってなるときはもちろんあります
      なので...
  3 : execute positioned とは
      これは execute at と違い単純です
    書き方
      execute positioned X座標 Y座標 Z座標
    どういう意味?
      簡単です
      これはさっきと違い
      X Y Z座標でコマンドを実行する位置を指定します
      たとえば
      execute positioned 0 60 0 run fill ~10 ~10 ~10 ~-10 ~-10 ~-10 air destroy
      の場合(真似すると重くなったり　建築物が壊れたりする恐れがあるので書かない方がいいです)
      座標 x0 y60 z0　で　20x20の範囲のブロックを破壊します
        豆知識 : fill destroy は　普通の fill と違いブロックを破壊した後に経験値やアイテムを落とす
  4 : execute store
      これは難しいです　理解するのに時間がかかりましたし　まだ僕もわかってないことだらけだとおもいます
    書き方
      execute store result entityまたはscoreまたはbossbar セレクター (NBT) (型) 1 run
    どういう意味?
      ※僕はsuccessの使い方を知りません
      ちなみに上の書き方は entityまたはscoreまたはbossbarの部分で entity を選んだ場合の書き方です
      プレイヤー以外のエンティティのNBTをコマンドの実行結果に置き換えます
      たとえば 
      data get entity セレクター Health
      の場合 実行結果は エンティティ(セレクター)の現在の体力になります
      あとついでに result score の説明も
      execute as セレクター store result score @s スコア名 run data get entity @s Health
      にすると
      セレクターのスコア(スコア名)がセレクターのHPになります
      store result score は
      セレクターの後にスコア名を書き。スコアに実行結果を代入するコマンドです
      そしてbossbarは今後説明していく予定です...
  5 : execute if と unless
      これはifとunlessを一緒に説明します
      理由は片方を説明すれば片方は一言で説明できるからです
    書き方
      execute if block または entity または score
      
      block と　書いた場合
        block X座標 Y座標 Z座標 ブロックID
      entity と 書いた場合
        entity セレクター
      score と 書いた場合
        score セレクター スコア名 下に書いてあるどれか セレクター2 スコア名2
        ここで = と書いた場合は　セレクターのスコア名 が セレクター2のスコア名2　同じ場合
        ここで < と書いた場合は  セレクターのスコア名 が セレクター2のスコア名2 より "少ない" 場合
        ここで > と書いた場合は  セレクターのスコア名 が セレクター2のスコア名2 より "多い" 場合
        ここで <= と書いた場合は  セレクターのスコア名 が セレクター2のスコア名2 より "少ない" 場合と同じ場合
        ここで >= と書いた場合は  セレクターのスコア名 が セレクター2のスコア名2 より "多い" 場合と同じ場合
     どういう意味?
       if の場合は条件が通っている場合場合
       unless の場合は条件が通っていない場合
       
       block の場合は 座標の　ブロック　が　ブロックID　と　同じ場合
       entity の場合は セレクターに"一致する"　エンティティ　がいた場合
       ※score は 上で説明しています
  応用編 : 1
       地面に立っている場合
       これは... NBTやコマンドをある程度知ってる人だと　は？　ってなりますw
       正直これはNBTのOnGroundで解決できる場合が結構あります
       ですが
       OnGroundで消す処理を行っていた場合
       水に浮いてるときにOnGroundはtrueにはなりません
       もしコマンドでアイテムを降らせていた場合
       OnGroundである程度時間がたったら消せばいいや
       だと...
       水の上にいるアイテムは全部消えません!!
       これは。。。下手したらクラッシュする可能性があります
       気を付けないと...
       そして　それじゃどうやって水に入ってることも検知するのさ！
       ってなった時
       
       execute as セレクター at @s unless block ~ ~-0.1 ~ air run kill
       
       こうすると...
       自分の0.1マス下のブロックが空気意外だった場合にkillします
       水でもkillされます
第二章 : NBT
       これを覚えるのは大変ですが。ものすごく便利です
       基本的に NBT は /dataコマンドで変更しますが
       execute store などで変更する場合もあります
       ここの章はよく使う NBT だけ書いていきます
       すべて書くのは不可能に近いからです
       知っている方も多いですがプレイヤーのNBTは変更できません
       そして 覚えてるものは毎回型を書きます
 1 : Health (double型)
       Entityの体力を管理しているものです
       たとえば /data modify entity @e[type=pig,limit=1,sort=nearest] Health set Value 1
       と打つと...
       一番近くの豚の体力が1になります
